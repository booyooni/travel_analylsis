# Online Retail Streamlit Dashboard Prompt

## 목적

아래 내용을 그대로 참고하여 `online-retail/data/online_retail.parquet` 데이터를 기반으로 Streamlit 종합 대시보드를 구현하라. 이 대시보드는 온라인 쇼핑몰 운영 담당자가 전체 거래 현황, 상품 성과, 고객 행동, 국가별 거래, 핵심 KPI, 리텐션, RFM 세그먼트까지 한 번에 파악할 수 있도록 설계되어야 한다.

결과물은 단순 EDA 화면이 아니라 실제 업무용 분석 대시보드여야 하며, 검색 기능과 목록 조회 기능까지 포함해야 한다.

모든 시각화는 반드시 Plotly로만 구현하라. matplotlib, seaborn, altair, bokeh 등 다른 시각화 라이브러리는 사용하지 마라.

## 데이터 소스

- 입력 파일: `online-retail/data/online_retail.parquet`
- 주요 컬럼
  - `InvoiceNo`: 주문번호, 취소 주문은 일반적으로 `C`로 시작
  - `StockCode`: 상품 코드
  - `Description`: 상품명
  - `Quantity`: 판매 수량
  - `InvoiceDate`: 주문 일시
  - `UnitPrice`: 상품 단가
  - `CustomerID`: 고객 ID
  - `Country`: 국가

## 구현해야 할 대시보드 목표

온라인 쇼핑몰 담당자 관점에서 아래 질문에 바로 답할 수 있어야 한다.

- 우리 쇼핑몰의 전체 거래 규모는 어느 정도인가
- 최근 월별 매출과 주문은 어떻게 변하고 있는가
- 어떤 국가에서 거래가 많이 발생하는가
- 어떤 상품이 많이 팔리고, 어떤 고객이 핵심 고객인가
- 신규 고객은 얼마나 유입되고 있는가
- 고객 재구매와 리텐션은 어떤 상태인가
- 고객을 RFM 기준으로 어떻게 구분할 수 있는가
- 특정 상품, 고객, 주문 정보를 빠르게 조회할 수 있는가

## 반드시 포함할 기능

### 1. 종합 KPI 섹션

최상단에 KPI 카드를 배치하라.

- 전체 데이터 수
- 전체 주문 수
- 전체 상품 수
- 전체 고객 수
- 총 매출
- 평균 주문금액 AOV
- 평균 고객당 매출 ARPPU 또는 ARPU 성격의 지표
- 취소 주문 수와 취소율
- 선택 기간 기준 신규 고객 수

### 2. 필터 영역

사이드바 또는 상단 필터로 아래를 제공하라.

- 날짜 범위 필터
- 국가 다중 선택
- 상품명 또는 상품코드 검색
- 고객 ID 검색
- 취소 주문 포함 여부
- 정제 데이터 사용 여부 토글

### 3. 전체 거래 현황 화면

다음 시각화를 포함하라.

- 일별 또는 월별 매출 추이 라인차트
- 일별 또는 월별 주문 수 추이
- 일별 또는 월별 활성 고객 수 추이
- 국가별 매출 막대차트
- 국가별 주문 수 또는 고객 수 비교 차트
- 국가별 매출 비중 또는 주문 비중을 보여주는 트리맵 또는 선버스트 차트
- 상위 상품 매출 차트
- 상위 상품 판매수량 차트

### 4. 상품 분석 화면

다음 기능을 포함하라.

- 상품 검색창
- 상품 목록 테이블
- 선택 상품 상세 카드
- 상품별 누적 매출, 판매수량, 주문 수, 구매 고객 수
- 월별 상품 매출 추이
- 국가별 해당 상품 판매 현황
- 상품 카테고리성 범주 비교가 가능하도록 상품별 또는 국가별 비중을 표현하는 트리맵 또는 선버스트 차트
- 상위 상품 랭킹 테이블

### 5. 고객 분석 화면

다음 기능을 포함하라.

- 고객 ID 검색창
- 고객 목록 테이블
- 선택 고객 상세 카드
- 고객별 총 구매금액, 총 주문 수, 평균 주문금액, 최근 구매일
- 고객별 구매 월별 추이
- 국가별 고객 분포
- 상위 고객 랭킹 테이블

### 6. 판매내역 조회 화면

다음 기능을 포함하라.

- 주문번호 검색
- 주문 단위 거래내역 테이블
- 고객 기준 거래내역 필터
- 상품 기준 거래내역 필터
- 국가 기준 거래내역 필터
- 데이터 다운로드 기능 CSV

### 7. 국가별 거래 현황 화면

다음 내용을 포함하라.

- 국가별 총매출, 주문 수, 고객 수 테이블
- 국가별 평균 주문금액
- 국가별 월별 매출 추이
- 상위 국가 비교 시각화
- 범주형 비교를 위한 국가별 트리맵 또는 선버스트 차트

### 8. 주요 지표 화면

아래 KPI를 월별 기준으로 계산하고 시각화하라.

- `DAU`: 일자별 고유 구매 고객 수
- `MAU`: 월별 고유 구매 고객 수
- `월별 ARPPU`: 해당 월 총매출 / 해당 월 구매 고객 수
- `월별 고객 획득 수`: 해당 월 첫 구매 고객 수
- `월별 주문 수`
- `월별 재구매 고객 수`
- `월별 재구매율`

각 지표는 정의를 화면에 짧게 설명하고, 차트와 테이블을 같이 제공하라.

### 9. 리텐션 분석 화면

코호트 기반 리텐션 테이블과 히트맵을 구현하라.

- 코호트 기준: 고객의 첫 구매 월
- 리텐션 측정 단위: 월
- `Month 0`은 첫 구매 월
- 각 코호트별로 이후 월의 재구매 고객 비율 계산
- 리텐션 테이블과 히트맵 동시 제공
- 선택 코호트 상세 고객 수 표시

### 10. RFM 분석 화면

RFM 분석을 구현하라.

- `Recency`: 기준일 대비 마지막 구매일 차이
- `Frequency`: 주문 수
- `Monetary`: 총 구매금액
- 분위수 기반 점수화 또는 명시적 구간화
- RFM 스코어와 세그먼트 부여
- 예시 세그먼트: Champions, Loyal Customers, Potential Loyalists, At Risk, Hibernating
- 세그먼트별 고객 수와 매출 기여도 시각화
- 세그먼트 비중을 직관적으로 볼 수 있는 트리맵 또는 선버스트 차트
- 고객 세그먼트 목록 조회 가능하도록 구현

### 11. 검색 및 목록 조회 기능

검색과 테이블 탐색은 반드시 실무적으로 쓸 수 있을 정도로 구현하라.

- 상품 검색: 코드, 상품명 부분일치
- 고객 검색: 고객 ID
- 주문 검색: 주문번호
- 결과 테이블 정렬
- 페이지네이션 또는 화면 성능을 고려한 일부 행 표시
- 선택 행 상세보기

## 시각화 및 설명 작성 규칙

모든 차트와 표는 단순 출력으로 끝내지 말고, 반드시 데이터 기준과 해석 가이드를 함께 제공하라.

- 모든 시각화는 Plotly로만 구현하라
- 범주형 데이터 시각화에는 막대차트뿐 아니라 treemap 또는 sunburst 차트를 적극 활용하라
- 각 차트 바로 아래에 `데이터 기준` 설명 블록을 추가하라
- 각 차트 바로 아래에 `해석 방법` 설명 블록을 추가하라
- 대시보드 전체 하단에도 공통 `데이터 기준 및 가공 방식` 섹션을 추가하라
- 대시보드 전체 하단에도 공통 `대시보드 해석 가이드` 섹션을 추가하라

각 설명 블록은 아래 조건을 반드시 만족해야 한다.

- `데이터 기준` 문구는 50자 이상으로 작성하라
- `해석 방법` 문구는 50자 이상으로 작성하라
- 설명에는 최소한 원본 행 수 또는 필터 적용 후 행 수, 사용 컬럼, 집계 기준, 전처리 또는 파생 컬럼 생성 규칙을 포함하라
- 설명에는 최소한 해당 차트에서 무엇이 높고 낮은지, 어떤 패턴을 봐야 하는지, 비교 시 주의할 점을 포함하라

각 시각화 설명의 예시는 아래 수준으로 구체적으로 작성하라.

- 데이터 기준 예시: 이 그래프는 정제된 거래 데이터 397,884행을 기준으로 `InvoiceDate`를 월 단위로 변환한 뒤 `Sales=Quantity*UnitPrice`를 합산하여 월별 매출을 계산한 결과이며, `Quantity<=0`, `UnitPrice<=0`, `CustomerID` 결측, `Description` 결측 행은 제외하였다.
- 해석 방법 예시: 선이 상승하는 구간은 월 매출이 증가한 시점이며, 특정 월의 급등은 시즌성이나 대량 주문 영향일 수 있으므로 주문 수와 활성 고객 수를 함께 비교해 실제 성장인지 일시적 집중인지 구분해서 해석해야 한다.

## KPI 정의 규칙

대시보드 전반에서 지표 정의를 일관되게 유지하라.

- 매출: `Quantity * UnitPrice`
- 주문 수: `InvoiceNo` 고유 개수
- 구매 고객 수: `CustomerID` 고유 개수
- 활성 고객 수: 해당 기간에 구매가 발생한 고유 고객 수
- 신규 고객: 해당 월이 첫 구매 월인 고객
- 재구매 고객: 해당 월 구매 고객 중 첫 구매 월보다 이후에도 구매한 고객
- ARPPU: 해당 기간 총매출 / 해당 기간 구매 고객 수
- AOV: 해당 기간 총매출 / 해당 기간 주문 수

취소 주문 처리 규칙도 명확히 반영하라.

- 기본 KPI는 정제된 양수 거래 기준으로 계산
- 원본 데이터 기준 현황도 별도 참고 가능하게 하라
- 취소 주문을 포함할지 제외할지 필터로 선택 가능하게 하라

## UX 및 화면 구성 요구사항

- Streamlit 기반 단일 앱으로 구현
- 사이드바 네비게이션 또는 탭 구조 사용
- 첫 화면은 종합 대시보드
- KPI 카드, 차트, 데이터 테이블이 균형 있게 보이도록 구성
- 숫자는 천 단위 구분기호로 포맷
- 한국어 라벨 사용
- 로딩 속도를 위해 `st.cache_data` 활용
- 데이터가 큰 편이므로 불필요한 재계산을 피하라
- 각 시각화 하단에 `데이터 기준`과 `해석 방법` 텍스트 영역을 항상 함께 배치하라
- 대시보드 맨 아래에는 전체 데이터 기준, 전처리 방식, 해석 유의사항을 정리한 설명 영역을 고정적으로 배치하라

## 구현 시 주의사항

- `CustomerID` 결측이 많으므로 고객 분석용 지표는 반드시 결측 제거 후 계산하라
- `Quantity <= 0` 또는 `UnitPrice <= 0` 값은 기본 분석에서 제외하라
- `InvoiceNo`가 `C`로 시작하는 취소 건은 별도 상태로 인식하라
- 일부 상품은 `Description`이 누락되어 있으므로 기본 상품 분석에서는 제외하거나 `Unknown` 처리 기준을 명확히 하라
- 극단값이 존재하므로 매출 차트는 상위 N 표시나 로그 스케일 여부를 고려하라
